Document Domain (General)

The document domain is the expected style or type of writing for your users and helps Grammarly offer appropriate writing suggestions. For instance, someone may want to use strictly formal writing for an academic paper, but when they're writing a poem, they want to be free to break grammar rules. By default, the Grammarly Text Editor Plugin has a document domain of "general", which provides a medium level of strictness that's applicable to a wide range of writing.

You can configure the Grammarly Text Editor Plugin to analyze text according to a different, more specific domain to get the most accurate and relevant writing suggestions for your users. On this page, the plugin is set to use the "general" domain setting.

Try the Grammarly Text Editor Plugin in the text area below!

You'll find that suggestions can possibly be different in different domains.

The configured domain cannot be overridden by your users from the settings menu.
